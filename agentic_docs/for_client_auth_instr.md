# Инструкция по интеграции сброса пароля на стороне клиента (Frontend)

Данный документ описывает, как реализовать интерфейс восстановления пароля на клиенте, взаимодействуя с эндпоинтами бэкенда.

---

## 1. Общий сценарий использования (User Flow)

1. Пользователь на странице входа нажимает на ссылку **"Забыли пароль?"**.
2. Переходит на страницу запроса сброса пароля, вводит свой email и отправляет форму.
3. Бэкенд проверяет email. Если пользователь найден, отправляет письмо с уникальной ссылкой на сброс. Пользователь видит сообщение: *"Инструкции отправлены на ваш email"*.
4. Пользователь переходит по ссылке из письма: `${FRONTEND_URL}/auth/reset-password?token=<TOKEN>`.
5. На фронтенде открывается страница сброса пароля. Из адресной строки извлекается query-параметр `token`.
6. Пользователь вводит новый пароль, подтверждает его и отправляет форму.
7. В случае успешного сброса пользователь перенаправляется на страницу входа.

---

## 2. Страница запроса сброса пароля (`ForgotPassword`)

### UI-элементы
- Поле ввода **Email** (с валидацией формата).
- Кнопка **"Отправить ссылку"**.

### Интеграция API
- **Метод**: `POST`
- **URL**: `/auth/forgot-password`
- **Тело запроса (JSON)**:
  ```json
  {
    "email": "user@example.com"
  }
  ```

### Особенности обработки ответов
- **Успешный ответ (HTTP 201)**:
  Покажите пользователю сообщение: *"Инструкции по восстановлению пароля отправлены на указанный email (если данный аккаунт зарегистрирован)"*.
  > **Важно**: Бэкенд всегда возвращает статус 201 (даже если email не существует в базе), чтобы злоумышленники не могли сканировать базу на наличие зарегистрированных email-адресов.
- **Ошибка (HTTP 400 и др.)**:
  Покажите ошибку валидации ввода.

## 3. Страница установки нового пароля (`ResetPassword`)

Эта страница должна быть доступна по пути `/auth/reset-password` (или по адресу, настроенному в переменной `FRONTEND_URL` на бэкенде).

### Получение токена
При переходе на эту страницу извлеките токен из query-параметров URL:
```javascript
// Пример на React / Vue / Vanilla JS
const urlParams = new URLSearchParams(window.location.search);
const token = urlParams.get('token');
```
Если токен отсутствует в URL, заблокируйте форму и покажите предупреждение: *"Неверная или неполная ссылка для сброса пароля"*.

### UI-элементы
- Скрытое поле или переменная состояния для хранения `token`.
- Поле ввода **"Новый пароль"** (минимальная длина: 6 символов).
- Поле ввода **"Подтвердите новый пароль"** (с проверкой на совпадение на клиенте).
- Кнопка **"Сохранить новый пароль"**.

### Интеграция API
- **Метод**: `POST`
- **URL**: `/auth/reset-password`
- **Тело запроса (JSON)**:
  ```json
  {
    "token": "значение_токена_из_url",
    "password": "новый_пароль_пользователя"
  }
  ```

### Особенности обработки ответов
- **Успешный ответ (HTTP 201)**:
  Покажите уведомление: *"Пароль успешно изменен!"* и перенаправьте пользователя на страницу входа (`/auth/login`).
- **Ошибка (HTTP 400 - Bad Request)**:
  Возникает, если токен недействителен или истекло время его жизни (1 час).
  Покажите пользователю сообщение: *"Ссылка для сброса пароля недействительна или устарела. Пожалуйста, запросите сброс пароля заново."* и предложите ссылку на возврат к странице `ForgotPassword`.

## 4. Пример реализации на TypeScript/React (Fetch API)

### Шаг 1: Запрос токена на Email
```typescript
async function handleForgotPassword(email: string) {
  try {
    const response = await fetch(`${process.env.API_URL}/auth/forgot-password`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email }),
    });

    const data = await response.json();
    alert(data.message || 'Инструкции отправлены на email.');
  } catch (error) {
    console.error('Ошибка:', error);
    alert('Произошла непредвиденная ошибка. Попробуйте позже.');
  }
}
```

### Шаг 2: Сброс пароля по токену
```typescript
async function handleResetPassword(token: string, password: string) {
  try {
    const response = await fetch(`${process.env.API_URL}/auth/reset-password`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ token, password }),
    });

    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(errorData.message || 'Не удалось сбросить пароль.');
    }

    alert('Пароль успешно обновлен. Теперь вы можете войти в систему.');
    // Редирект на логин
    window.location.href = '/auth/login';
  } catch (error: any) {
    alert(`Ошибка: ${error.message}`);
  }
}
```
