# 📱 ТЗ для ИИ-агента: Групповая Telegram-рассылка во Frontend (Admin UI)

Привет, агент! Твоя задача — реализовать интерфейс массовой (групповой) рассылки сообщений студентам через Telegram в **панели управления администратора (Admin UI)**. 

Этот функционал должен быть бесшовно интегрирован в существующую таблицу студентов.

---

## 1. UI/UX Сценарий рассылки

### 1.1 Флоу администратора
1. **Выбор адресатов:**
   * В таблице студентов администратор отмечает галочками (checkboxes) нужных студентов.
   * На панели действий над выбранными строками появляется кнопка **"Написать в Telegram"**.
2. **Форма написания сообщения (Modal/Drawer):**
   * При клике открывается модальное окно или выдвижная панель (Drawer).
   * Отображается информация о количестве получателей: *"Выбрано студентов: 15"*.
   * **Поле ввода текста** (TextArea) с поддержкой форматирования и подсказкой о разрешенных HTML-тегах:
     * `<b>текст</b>` — жирный
     * `<i>текст</i>` — курсив
     * `<code>текст</code>` — моноширинный
   * Максимальная длина сообщения — **4000 символов** (лимит Telegram). Показывается счетчик символов.
3. **Отправка:**
   * Кнопка **"Отправить рассылку"**. При клике показывается Spinner.
   * UI делает запрос `POST /organizations/:orgId/telegram/broadcast`.
4. **Отображение результата (Toast/Alert):**
   * Получив успешный ответ `200 OK`, UI закрывает модалку и показывает Toast-сообщение с детальной статистикой:
     * *"Рассылка успешно запущена на Bot Server! Сообщение будет доставлено 12 из 15 выбранных студентов."*
     * Если разница есть (3 студента не получат), вывести примечание: *"3 студента не привязали свои Telegram-аккаунты в личном кабинете."*

## 2. API Контракты и TypeScript типы

Используй эти интерфейсы и эндпоинты для реализации API-клиента на фронтенде.

### 2.1 TypeScript интерфейсы

```typescript
// Запрос на рассылку
export interface SendTelegramBroadcastRequestDto {
  userIds: string[]; // Массив UUID пользователей (НЕ студентов, а их userId)
  text: string;      // Текст сообщения (HTML)
}

// Ответ от Main Server
export interface SendTelegramBroadcastResponseDto {
  success: boolean;       // Успешность постановки задачи в очередь
  sentCount: number;      // Количество студентов, у которых найден привязанный TG
  totalRequested: number; // Сколько студентов было изначально выделено в UI
  message?: string;       // Поясняющий текст от сервера
}
```

### 2.2 Спецификация REST API эндпоинта

- **Метод**: `POST`
- **Путь**: `/organizations/:orgId/telegram/broadcast`
- **Заголовки**: `Authorization: Bearer <Admin_JWT>`
- **Request Body**: `SendTelegramBroadcastRequestDto`
- **Response (200 OK)**: `SendTelegramBroadcastResponseDto`

## 3. Обработка ошибок во Frontend

При интеграции с API обработай следующие сценарии сбоев:

- **`400 Bad Request`**: Пустой массив пользователей или пустое сообщение.
  * *Решение*: Валидируй форму на клиенте (делай кнопку "Отправить" неактивной, если TextArea пуст).
- **`401 Unauthorized`**: Истек сеанс администратора.
  * *Решение*: Стандартный редирект на логин.
- **Ответ `success: true, sentCount: 0`**: Ни один из выбранных студентов не привязал Telegram.
  * *Решение*: Выведи предупреждающий Alert: *"Ни один из выбранных студентов не подключил Telegram-уведомления. Рассылка отменена."*
